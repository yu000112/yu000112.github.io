DBMS(数据库管理系统)–执行–> SQL(结构化查询语言)–操作–>DB(数据库)

  • SQL是一套标准，程序员主要学习的是SQL语句，这个SQL在mysql中可以用，在oracle中也可以用。

  • net start mysql(开启) net stop mysql(关闭) 必须用管理员身份运行cmd

    • 其他服务的启停都可以采用以上命令 net start/stop 服务名称

  • 端口号：端口号是任何一个软件/应用都会有的，端口号是应用的唯一代表。端口号通常和IP地址在一块，IP地址用来定位计算机，端口号port用来定位计算机上某个服务/应用的。在同一台计算机上，端口号不能重复，具有唯一性。

  • mysql数据库启动的时候，此服务占有的默认端口号是3306

  • 使用客户端登陆mysql数据库 mysql -uroot -p密码

    退出 exit

  • 查看mysql中有哪些数据库？ show databases;

    查看数据库下有哪些表？ show tables;

    查看表中的数据 select * from 表名; （效率低，可读性差）

    不看表中的数据，只看表的结构 desc 表名;

    选择使用数据库 use 数据库名字;

    查看数据库的版本号 select version();

    创建数据库 create database 数据库名字; (命令可以不区分大小写 )

    新建数据库–>use数据库–>导入表

  • 数据库的最基本单元是表：table ,因为表比较直观

    • 行（row）:被称为数据/数据
    • 列（column）:被称为字段

  • SQL 语句分类 DQL(select) DML(insert delete update) DDL(create drop alter) TCL (commit rollback) DCL(grant revoke)

SQL语法

1、查询 （select 不会进行修改操作，只负责查询）

  • 普通查询

select 字段名1,字段名2 from 表名; (如果是多个字段，则这样写；一个字段就写一个即可)

as 可以起别名，也可以省略as

select 1,2 22 from biaoming; (对as前面的那个名字起别名)

select dep ‘dep 1’ from biaoming;

字段可以参与运算

select ename,sal*12 as ‘yealsal’ from emp;

  • 条件查询

select

​ 字段1，字段2，字段3…

from

​ 表名

where

​ 条件;

mysql> select empno,ename,sal from emp where sal >=2450 and sal <= 3000;

等于select empno,ename,sal from emp where sal between 2450 and 3000;

数据库中不能用=衡量null，需要用 is null/is not null，因为数据库中地null代表什么也没有，不是一个值。

mysql> select empno,ename,job,sal from emp where job = ‘manager’ and sal > 2500;

and 优先级大于 or

is null / is not null in / not in

mysql> select empno,ename,job,sal from emp where job in(‘manager’, ‘salesman’);

等于select empno,ename,job,sal from emp where job = ‘manager’ or job = ‘salesman’;

找出名字中含有o的：

mysql> select ename from emp where ename like ‘%o%’;

找出名字以t结尾的：

mysql> select ename from emp where ename like ‘%t’;

找出名字以k开始的：

mysql> select ename from emp where ename like ‘k%’;

找出第二个字母是a的:

mysql> select ename from emp where ename like ‘_a%’;

找出名字中有“_”的：

select ename from emp where ename like ‘%\_%’;

  • 排序

select ename,sal from emp order by sal; (默认升序，从小到大)

select ename,sal from emp order by sal desc; (指定降序，从大到小)

select ename,sal from emp order by sal asc; (指定升序)

select ename,sal from emp order by sal asc, ename asc; (按照薪资升序，如果薪资一样，按照名字升序)

  • 函数

    • 数据处理函数（单行处理函数：一个输入对应一个输出）

      select substr(ename,1, 1) as ename from emp; (起始下标从1开始)

      select ename from emp where ename like ‘A%’;

      = select ename from emp where substr(ename,1,1) = ‘A’;

      select round(1234.456, 1) as result from emp;

      select rand() from emp; (生成随机数)

      select round(rand()*100, 0) from emp;

      concat(,)函数可以拼接字符串

      有null涉及的运算结果就是null，为了避免这个现象，需要使用ifnull(,)函数

      case..when..then..when..then..else..end;

      select ename,job,sal as oldsal, (case job when ‘manager’ then sal * 1.1 when ‘salesman’ then sal * 1.5 else sal end) as newsal from emp;

    • 分组函数（多行处理函数）

      count() max() sum() avg()

      分组函数自动忽略null

      cout(具体字段) ：表示统计该字段下所有不为null的元素的总数

      cout(*)：统计表中的总行数

      分组函数不能直接用在where子句中

  • 分组查询

    select … from … group by …

    组合 select … from … where … group by … having … order by … limit…

    关键字执行循序: from –> where –> group by –> having –> select –> order by –> limit

    select job, sum(sal) from emp group by job;

    在一条select语句中，如果有group by 语句的话，select后面只能跟：参加分组的字段，以及分组函数，其他的一律不能跟。

    having只能和group by 联合使用。having可以对分完组之后的数据进一步过滤

    select deptno, max(sal) from emp group by deptno having max(sal) > 3000;

    select deptno, max(sal) from emp where sal > 3000 group by deptno;

    select distinct job,deptno from emp; (除去job和deptno完全一样的)

    select count(distinct job) from emp;

  • 连接查询

    怎么避免笛卡尔积现象？连接时加条件，满足这个条件的记录被筛选出来。

    • 内连接之等值连接

    select e.ename,d.dname from emp e,dept d where e.deptno = d.deptno; (SQL92语法)

    select e.ename,d.dname from emp e join dept d on e.deptno = d.deptno; (SQL99语法)

    • 内连接之非等值连接

    select e.ename,e.sal,s.grade from emp e join salgrade s on e.sal between s.losal and s.hisal；

    • 内连接之自连接

    select a.ename as ‘员工名’ ,b.ename as ‘领导名’ from emp a join emp b on a.mgr = b.empno;

    内连接的表是平等的

    • 右外连接

    select e.ename,d.dname from emp e right join dept d on e.deptno = d.deptno;

    right表示将join关键字右边的这张表看成主表，主要是为了将这张表的数据全部查出来，捎带着关联查询左边的表

    • 左外连接

    select e.ename,d.dname from dept d left join emp e on e.deptno = d.deptno;

    • 多表联查

    select e.ename,e.sal,d.dname,s.grade from emp e join dept d on e.deptno = d.deptno join salgrade s on e.sal between s.losal and s.hisal left join emp l on e.mgr = l.empno;

    • 子查询

      select …(select) from …(select) where …(select)

      select ename,sal from emp where sal > (select min(sal) from emp);

      技巧：from 后面的子查询，可以将子查询当作一张临时表。

      select t.*,s.grade from (select job,avg(sal) as avgsal from emp group by job) t join salgrade s on t.avgsal between s.losal and s.hisal;

      select e.ename,d.dname from emp e join dept d on e.deptno = d.deptno;

      = select e.ename, (select d.dname from dept d where e.deptno = d.deptno) as dname from emp e;

  • union合并查询结果集(union 效率高，把乘法变成加法)

    select job,ename from emp where job = ‘manager’ or job = ‘salesman’;

    = select job,ename from emp where job = ‘manager’ union select job,ename from emp where job = ‘salesman’;

  • limit（将查询结果集的一部分取出来，通常使用在分页查询中）

    select ename,sal from emp order by sal limit 0,5; (0表示起始位置，5表示长度)

    • 分页

      每页显示pageSize条记录

      第pageNo页：limit (pageNo - 1) * pageSize , pageSize

  • format 格式化数字 format(数字，‘格式’)

    select ename,format(sal,’$999,999’) as sal from emp;

  • str_to_date:将字符串varchar类型转换成date类型

    %Y %m %d %h %i %s 年月日时分秒

    %Y-%m-%d

  • date_format:将date类型转换成具有一定格式的varchar 字符串类型

    date_format(日期数据名称, ‘日期格式’)

    select id,name,date_format(birth,’%m-%d-%Y’) as birth from t_user;

  • 表的创建

    • (DDL: create drop alter)

      create table 表名 (字段名1 数据类型，字段2 数据类型，字段3 数据类型);

      表名建议以t_ 或 tbl_开始，可读性强

      create table t_user(birth date,name varchar(20),sex char(1));

      mysql 中的数据类型：常见的 int(11) double varchar char() bigint float date datetime clob blob datetime date

      括号里面的数字是长度（数字或中文一个字算一个长度）

      varchar 动态分配空间， char(255)不需要动态分配空间，速度快

      date 短日期类型 datetime 长日期类型 clob 字符大对象 最多可存储4G的字符串，比如存储一篇文章,超过255个字符都要用clob blob 二进制大对象*

      • 快速创建表(把一个查询结果当作一张表新建)

        create table emp2 as select * from emp;

        create table mytable as select empno,ename from emp where job = ‘manager’;

  • 删除表 drop

    drop table if exists t_student;

  • 插入 insert

    一次可以插入多条记录

    insert into 表名(字段名1，字段名2，字段名3…) values (值1，值2，值3…);

    字段名和值一一对应

    insert into t_student(no,name,sex,age,email) values (1, ‘zhangsan’, ‘m’, 20, ‘zhangsan@123.com‘);

    sex char(1) default ‘m’, （default 可以指定默认值）

  • update 修改(DML)

    update 表名 set 字段名1=值1，字段名2 = 值2，字段名3 = 值3….where 条件；

    没有条件限制会将所有数据更新

    update t_user set name = ‘jack’ ,birth = ‘2000-10-11’,

  • delete 删除表的数据,支持回滚（rollback;），数据占用的空间不会被释放，效率低。

    delete from t_user;

    delete from t_student where no = 1;

  • truncate 删除表的数据效率高，表被一次截断，不支持回滚，快速。

    truncate table t_student2;

  • 约束（not null unique primary key foreign key）

    • 列级约束

      not null只有列级约束

      create table t_vip(id int,name varchar(255) unique,email varchar(255) unique);

    • 表级约束

    create table t_vip(id int,name varchar(255), email varchar(255), unique(name,email)); //复合约束

    create table t_vip(id int, name varchar(255), primary key(id));

    create table t_vip(id int, name varchar(255), primary key(id,name)); //复合主键

    • 主键约束

      • 主键约束

      • 主键字段

      • 主键值（每一行记录的唯一标识，身份证号） ：任何一张表都应该有主键值，没有主键，表无效,一张表，主键约束只能添加1个 not null unique

        drop table if exists t_vip;

        create table t_vip(id int primary key auto_increment, name varchar(255));

        insert into t_vip(id,name) values(1,’zhangsan’);

        insert into t_vip(id,name) values(2,’lisi’);

        auto_increment 表示自增，从1开始，以1递增

    • 外键约束

      注意: t_class是父表 t_student是子表

      先删子再删父 先创父再创子 数据先插入父，再插入子

      create table t_student(

      no int primary key auto_increment, 
name varchar(255),
cno int,
foreign key(cno) references t_class(classno)

      );

    外键值可以为null 被引用的字段不一定是主键，但至少具有unique约束

  • 事务

    start transaction;

    commit;

    select @@transaction_isolation;

    set global transaction isolation level read uncommitted;

    set global transaction isolation level read committed;

    set global transaction isolation level repeatable read;

    set global transaction isolation level read serializable;

  • 索引(索引是各种数据库优化的重要手段，优化的时候优先考虑的因素就是索引)

    create index emp_ename_index on emp(ename);

    drop index emp_ename index on emp;

    explain select * from emp where ename = ‘king’; (看type)

    • 索引失效

      • select * from emp where ename like ‘%T’;

      • 使用or要求or两边的条件字段都要有索引，才会走索引。尽量少用or

      • 使用复合索引时，没有使用左侧的列查找，索引失效

        create index emp_job_sal_index on (job,sal);

        explain select * from emp where job = ‘manager’; //可以

        explain select * from emp where sal = 800; //不可

      • 在where中索引列 参加了运算，索引失效

        create index emp_sal_index on emp(sal);

        explain select * from emp where sal + 1 = 800; //失效

      • where当中索引列使用了函数

        explain select * from emp where lower(ename) = ‘smith’;

  • 视图(通过对视图的操作，会影响原表数据)

    视图对应的语句只能是DQL(select )语句，但是视图对象创建完成后可以对视图对象进行增删改查等操作

    create table dept2 as select * from dept;

    create view dept2_view as select * from dept2;

    drop view dept2_view;

    select * from dept2_view;

    insert into dept2_view(deptno,dname,loc) values(60,’sales’, ‘beijing’);

    create view emp_dept_view as select e.ename,e.sal,d.dname from emp e join dept d on e.deptno = d.deptno;
